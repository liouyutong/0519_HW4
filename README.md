#### 🔍 核心洞察與換模型原因總結
我們共研究了兩套不同的模型架構：
1. **模型 1 (MobileNetV2)**: 基於深度學習 (CNN) 架構。[參考來源](https://github.com/penny0411/HW4)
2. **模型 2 (HOG + SVM + 雙重驗證)**: 基於傳統機器學習與 OpenCV 影像處理技術。[參考來源](https://github.com/chhcgrrace/hw4)

**技術決策：為什麼最終採用測試準確率較低的「模型 2」？**
在靜態測試集中，模型 1 分數較高；但在我們實際用 Camera 進行 Demo 時，發現模型 1 存在嚴重的**過度自信**問題：由於訓練資料缺乏「Error」類別的圖片，深度學習模型容易強行將未知的畫面與背景歸類為剪刀、石頭或布。

為了解決這個問題，我們引用的**模型 2** 在預測流程中加入了**「膚色過濾」與「指縫數量計算 (Convexity Defects)」的雙重防呆驗證機制**。這補足了訓練集的缺陷，只要手勢指縫數量不符邏輯或預測機率偏低，系統就會自動將其歸類為 **"Error"**。這項機制大幅提升了實機測試的強健性 (Robustness)，幫助我們順利且精準地完成了 10 次包含錯誤手勢的 Demo 錄影。
